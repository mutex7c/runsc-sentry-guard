# Contributing to runsc-sentry-guard

Thank you for your interest in contributing! Because this daemon operates at the host edge 
with elevated privileges, we enforce strict engineering standards for all code submissions.

### The Golden Rule: Atomic Contributions
* **One PR = One Issue:** Pull requests must address a single, specific issue.
* **No Sprawling Refactors:** Unsolicited structural changes, "drive-by" optimizations, or formatting sweeps included alongside feature work will be immediately rejected. Keep your commits granular and strictly scoped to the ticket.
* **Open an Issue First:** If you want to introduce a new feature or architectural change, open a granular issue to discuss it *before* writing code. Unsolicited, large-scale PRs will not be merged.
